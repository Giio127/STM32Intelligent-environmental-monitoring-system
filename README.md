# STM32F103 多传感器环境监测系统 — 设计文档

## 1. 项目概述

基于 STM32F103C8T6 的多传感器环境监测系统，集成 DHT11 温湿度、数字光敏、水位模拟传感器，通过 0.96 寸 OLED 实时显示，支持本地按键与串口远程双模控制，具备阈值报警和声光提示功能。

**核心工具**：Keil MDK5、STM32 标准库、单总线协议、ADC 单通道采集、I2C 驱动、UART 串口通信、定时器中断、按键扫描与状态机。

## 3. 硬件连接与引脚分配

| 模块 | STM32引脚 | 功能 |
|------|----------|------|
| DHT11 DATA | PA1 | 单总线数据 |
| 水位传感器 AO | PA0 | ADC1_CH0 模拟输入 |
| 光敏传感器 DO | PA2 | GPIO输入（高=亮，低=暗） |
| OLED SCL | PB8 | I2C1_SCL |
| OLED SDA | PB9 | I2C1_SDA |
| 蜂鸣器 | PA8 | GPIO输出（低电平报警，有源蜂鸣器） |
| LED绿 | PB13 | GPIO输出（运行指示，高电平亮） |
| LED红 | PB12 | GPIO输出（报警指示，高电平亮） |
| 按键1 | PB11 | GPIO输入（功能键，低电平有效） |
| 按键2 | PB1 | GPIO输入（确认键，低电平有效） |
| UART_TX | PA9 | USART1_TX |
| UART_RX | PA10 | USART1_RX |

**引脚分配原则**：尽量使用连续引脚，避免冲突，I2C 固定在 PB8/PB9。

## 4. 按键状态机与系统模式

系统有三种工作模式，通过按键切换：

```
┌──────────┐   按键1短按  ┌──────────┐   按键1长按  ┌──────────┐
│  正常监测 │ ──────────→ │  参数设置 │ ──────────→ │  正常监测 │
│  模式    │ ←────────── │  模式    │ ←────────── │  模式    │
└──────────┘              └──────────┘              └──────────┘
     │                         │
     │ 按键1短按(报警时)        │ 按键2长按切换设置项
     ↓                         ↓
┌──────────┐              ┌──────────┐
│  报警管理 │              │  设置项  │
│  模式    │              │  切换    │
└──────────┘              └──────────┘
```

**按键扫描实现**：使用状态机实现非阻塞按键扫描，支持短按和长按检测（长按阈值 1000ms）。

### 按键功能定义

- **按键1短按**：正常模式下进入设置模式；设置模式下阈值+1；报警管理模式下返回正常模式
- **按键1长按**：设置模式下返回正常模式
- **按键2短按**：设置模式下阈值-1；报警管理模式下解除报警
- **按键2长按**：设置模式下切换设置项（温度 → 湿度 → 水位）

### 参数设置模式

按键2长按循环切换设置项（温度阈值 → 湿度阈值 → 水位阈值），按键1短按+1，按键2短按-1。

### OLED显示

- **正常模式**：显示 T:25℃ H:60% 光:亮 水位:50% + 系统状态
- **设置模式**：显示当前设置项和阈值
- **报警管理**：显示哪个参数报警 + 解除操作提示

## 5. 串口协议设计

**波特率**：115200bps，8N1

系统支持两种协议模式：HEX帧协议和文本命令协议，自动识别。

### 5.1 HEX帧协议

#### 帧格式

```
┌──────┬──────┬──────┬──────┬──────┬──────┐
│ 帧头 │ 地址 │ 命令 │ 数据 │ 校验 │ 帧尾 │
│ 0xAA │ 0x01 │ 1B   │ N B  │ XOR  │ 0x55 │
└──────┴──────┴──────┴──────┴──────┴──────┘
```

#### 命令字定义

| 方向 | 命令码 | 功能 | 数据内容 |
|------|--------|------|----------|
| 查询 | 0x01 | 查询温度 | 无 |
| 查询 | 0x02 | 查询湿度 | 无 |
| 查询 | 0x03 | 查询水位 | 无 |
| 查询 | 0x04 | 查询光照 | 无 |
| 查询 | 0x05 | 查询全部参数 | 无 |
| 控制 | 0x10 | 开启系统 | 无 |
| 控制 | 0x11 | 关闭系统 | 无 |
| 控制 | 0x12 | 确认解除报警 | 无 |
| 设置 | 0x20 | 设置温度阈值 | 1B: 阈值 |
| 设置 | 0x21 | 设置湿度阈值 | 1B: 阈值 |
| 设置 | 0x22 | 设置水位阈值 | 1B: 阈值 |
| 响应 | 0x80+原命令 | 应答 | 对应数据 |
| 报警 | 0xF0 | 主动上报报警 | 报警类型+温度+湿度+水位 |

#### 校验方式

地址 ^ 命令 ^ 数据 所有字节异或

#### 超时处理

接收帧超时 50ms 则丢弃当前帧，重新等待帧头。

### 5.2 文本命令协议

支持 ASCII 文本命令，以回车或换行结尾：

| 命令 | 功能 | 响应示例 |
|------|------|----------|
| `T?` | 查询温度 | `Temp=25C` |
| `H?` | 查询湿度 | `Humi=60%` |
| `W?` | 查询水位 | `Water=50%` |
| `L?` | 查询光照 | `Light=Bright` |
| `ALL?` | 查询全部 | `T:25C H:60% W:50% L:Bri` |
| `ON` | 开启系统 | `System ON` |
| `OFF` | 关闭系统 | `System OFF` |
| `CLR` | 解除报警 | `Alarm Cleared` |
| `T=35` | 设置温度阈值 | `TempTh=35C OK` |
| `H=80` | 设置湿度阈值 | `HumiTh=80% OK` |
| `W=70` | 设置水位阈值 | `WaterTh=70% OK` |

### 5.3 串口数据上报

系统每 1 秒自动上报一次传感器数据，格式：
```
T:25C H:60% W:50% L:Bri | Th:T30 H70 W50 | OK
T:38C H:85% W:75% L:Dar | Th:T30 H70 W50 | ALARM:T H W
```

## 6. OLED 显示布局

0.96 寸 OLED 128x64 像素，使用 8x16 字体（每字符 8px 宽，16px 高），可显示 4行×16列。

### 正常监测模式

```
┌────────────────┐
│ T:25/30C       │  行1: 温度当前值/阈值
│ H:60/70%       │  行2: 湿度当前值/阈值
│ W:50/50% L:Bri │  行3: 水位/阈值 + 光照
│ S:Run  A:None  │  行4: 系统状态 + 报警
└────────────────┘
```

### 参数设置模式

```
┌────────────────┐
│[Temp] Threshold│  行1: 当前设置项标题
│  Set:35 C      │  行2: 设置的阈值
│  Now:25 C      │  行3: 当前传感器实时值
│K1:+ K2:- Nxt  │  行4: 操作提示
└────────────────┘
```

### 报警管理模式

```
┌────────────────┐
│== ALARM MGMT ==│  行1: 标题
│T too HIGH!     │  行2: 报警项（轮显）
│T:38 > Th:35    │  行3: 当前值 > 阈值
│K2:Clr   K1:Back│  行4: 操作提示
└────────────────┘
```

**显示刷新**：主循环每 200ms 刷新一次数据区域，报警时立即刷新状态区域。

**报警状态显示**：当报警触发时，正常模式下行4显示 "A:T OVER!" 等提示，同时蜂鸣器和 LED 动作。报警管理模式下多个报警项轮流显示（每 2 秒切换）。

## 7. 报警机制

### 报警触发条件（任一满足即触发）

- 温度 > 设置的高阈值（默认 30℃）
- 湿度 > 设置的高阈值（默认 70%）
- 水位 > 设置的高阈值（默认 50%）

### 报警响应（同时执行，响应时间 < 100ms）

1. 蜂鸣器持续鸣叫（低电平驱动，有源蜂鸣器）
2. 红色 LED 常亮，绿色 LED 灭
3. OLED 状态行显示报警参数
4. 串口发送 0xF0 报警上报帧

### 报警解除

- 按键2手动确认解除，或
- 串口发送 0x12 命令解除
- 解除后：蜂鸣器停止、红 LED 灭、OLED 恢复 NORMAL 状态

### 报警优先级

多个参数同时报警时，OLED 轮流显示各报警项（每 2 秒切换），蜂鸣器和 LED 保持。

### 阈值默认值

| 参数 | 默认高阈值 | 范围 |
|------|-----------|------|
| 温度 | 30℃ | 0-100℃ |
| 湿度 | 70% | 0-100% |
| 水位 | 50% | 0-100% |

阈值可通过按键或串口修改，运行时保存在 RAM 中，断电恢复默认。

## 8. 主循环与定时器分配

### 定时器分配

- **TIM2**：用于精确延时（DHT11 单总线时序需要微秒级延时）
- **SysTick**：系统滴答定时器，提供 ms 级延时和时间基准

### DHT11 时序

单总线协议需要严格的微秒级时序，使用 TIM2 计数器实现精确延时，确保数据读取可靠。

### ADC 采集

单次转换模式，采集 PA0 通道，12 位精度，换算为 0-100% 水位百分比。
